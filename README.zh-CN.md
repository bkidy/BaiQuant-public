# 我把一个证券账户交给了 Codex

[English README](README.md) | [中文故事版 / Codex Series 01](README.zh-CN.md)

这是我准备写的 **Codex 系列文章第一篇**，也是 BaiQuant 这个项目的起点。

完整复盘：[Codex 系列 01：我把一个 A
股账户交给 AI，一个月后收益翻倍](https://glbai.com/posts/codex-built-a-share-quant-strategy/)

我想探索一个很具体的问题：

> 当我自己并不真正懂股票，也说不清每一只股票上涨或下跌背后的产业逻辑时，Codex
> 能不能帮我拓宽对金融市场的认知，快速验证交易想法，并且在真实账户里创造一些
> 可复盘的收益？

这个问题听起来有点冒险。它不是“AI 能不能预测股市”这么简单，也不是“写一个量化
框架”这么工程化。对我来说，它更像是一次关于 AI 上限的实验：把一个我并不熟悉、
但反馈极快、风险真实、数据足够多的领域交给 Codex，看看它到底能帮我走多远。

## 这个实验为什么开始

我并不是金融从业者，也不是传统意义上的股票高手。

我不知道每一家公司上涨时，市场到底在交易什么逻辑；也不知道下跌时，哪些是情绪，
哪些是基本面，哪些只是资金短期切换。很多时候，我能感受到市场很热，但我不知道
热在哪里；我能看到一只股票涨了很多，但不知道它是不是已经到了风险更高的位置。

所以我给 Codex 的目标非常宽泛：

> 帮我沉淀一套适合我当下交易习惯和风险预期的量化选股策略。

我没有一开始就告诉它固定公式，也没有让它照搬现成策略。我只告诉它我想要什么：

- 我主要想做选股；
- 我只能在收盘后更新数据；
- 我会手动交易，不需要全自动下单；
- 我希望它通过持续回测和复盘，找到一套更适合我的规则；
- 我愿意准备一个真实账户，尽量 100% 信任它给出的决策。

换句话说，我不是让 Codex 帮我写一个玩具项目，而是把它放进一个真实、有约束、
有损益反馈的场景里。

## 数据从哪里来

量化回测绕不开数据。

一开始我也不知道个人量化应该用什么数据源，于是我先让 GPT 帮我调研：哪些 A 股
数据源适合个人研究，价格不要太贵，覆盖要足够完整，最好能本地落库，方便反复跑。

最后我选择了 Tushare。

我把 token、数据接口、股票池、日线数据、复权方式、SQLite 落库方式都交给 Codex，
让它自己把数据链路搭起来。这个过程里，它不仅写了拉取数据的命令，也逐步处理了
限流、断点续传、SQLite 存储、全 A 股票池、复权价格、回测窗口这些非常实际的问题。

这一步对我很关键。因为如果没有真实数据，所有策略讨论都只是聊天；有了数据，
Codex 才能把“感觉”变成“可验证的实验”。

## 我给 Codex 跑了一个长目标

项目真正进入状态，是从 2026 年 5 月 23 日前后开始的。

我给了 Codex 一个比较长的 goal，让它围绕“帮我写出一套能用于盘后选股和实盘复盘
的量化策略”持续工作。它大概跑了 24 小时多一点：一边写代码，一边回测，一边根据
失败结果继续调整。

这个过程里，它做的事情包括：

- 搭建本地 A 股数据管线；
- 写股票筛选、因子计算、交易计划生成；
- 加入 A 股约束，比如 T+1、100 股一手、涨跌停、ST、停牌、流动性；
- 对不同策略做 2024、2025、2026 的历史验证；
- 反复检查为什么某些策略滞后、追高、回撤过大或者空仓；
- 最后保留更适合我手动交易节奏的公开策略 `steady-20d`。

这不是一次“让 AI 一步到位写出完美策略”的过程。更真实的情况是：它会犯错，会过拟合，
会选出我觉得太高的票，也会在行情变化时显得滞后。关键在于，每一次失败都能被写成
规则、代码、回测和复盘，然后进入下一轮迭代。

这就是我理解的 Harness 风格：

```text
一个模糊目标
  -> 拆成明确规则
  -> 写成可运行代码
  -> 用真实历史数据回测
  -> 找到失败原因
  -> 修改或删除规则
  -> 再跑一遍
```

Codex 的价值不只是“写代码快”，而是它能把一个模糊的认知问题，推进成一个持续验证
的实验系统。

## 真实账户里的反馈

我准备了一个账户，放了一些钱进去，然后决定尽量信任 Codex。

从 5 月 23 日开始的这段实验窗口里，市场本身确实很强，2026 年上半年 A 股行情也
给了很多策略发挥空间。这个前提必须说清楚，因为如果只展示收益，不讲环境，很容易
把行情红利误认为策略能力。

但即便如此，这次实验对我仍然有价值。

在那段震荡和主线快速切换的过程中，Codex 选出来的一些股票，确实帮助我的账户取得了
远超我原本预期的阶段性表现。它不是每天都对，也不是每一笔都赚钱，但它让我第一次
感受到：AI 不只是能回答“这个股票怎么样”，它还能把选股、回测、持仓、风险、复盘
放进同一个流程里，持续把我的判断上限往上抬。

我不会把这段收益包装成“AI 稳定赚钱”。它可能有运气成分，也可能受益于 2026 年的
市场环境。但我愿意继续做下去，因为它已经证明了一件事：

> Codex 可以在一个我不熟悉的领域里，帮我更快建立问题框架、验证假设、吸收知识，
> 甚至把这些验证结果连接到真实账户反馈里。

这就是我觉得值得记录和开源的原因。

## BaiQuant 是什么

BaiQuant 是这次实验沉淀出来的代码仓库。

它不是一个成熟的资管系统，也不是一个保证收益的自动交易机器人。它更像是一套个人
量化研究工作台：

| 模块 | 作用 |
| --- | --- |
| 数据管线 | 从 Tushare 等数据源拉取 A 股数据，并落到本地 SQLite |
| 选股策略 | 根据市场宽度、热点、技术指标、流动性和风险规则筛选股票 |
| 回测工具 | 对策略做历史验证，观察收益、回撤和失败窗口 |
| 盘后计划 | 收盘后生成次日候选股票和操作建议 |
| 本地 Web UI | 查看观察池、计划、持仓和复盘信息 |
| 交易记录 | 手动记录真实买卖，用来对齐策略和账户结果 |

当前公开保留的策略叫 **`steady-20d`**。它围绕大约 20 个交易日的持仓周期，结合
市场状态、行业强度、技术排名、流动性过滤和风险控制，生成一个更适合人工复核的
候选池。

## 为什么要开源

开源这个项目，不是因为我认为它已经是一个完美策略。

恰恰相反，我想把它作为一个样本：一个普通开发者如何把 Codex 放进一个陌生领域，
用 AI、数据、回测和真实反馈，快速探索一个方向的上限。

我希望这个仓库能留下这些东西：

- AI 如何把模糊需求变成可执行规则；
- Codex 如何参与金融研究和量化策略工程；
- 一个个人 A 股账户如何做盘后选股和复盘；
- 策略为什么不能只看收益，还要看回撤、交易约束和市场环境；
- 人和 AI 如何通过持续验证一起学习。

如果你关心的是“Codex 到底能不能在真实工作流里产生价值”，这个项目可能比单纯的
demo 更有参考意义。

## 快速试跑

创建环境并安装：

```bash
python3 -m venv .venv
.venv/bin/pip install -e ".[dev]"
```

运行测试：

```bash
.venv/bin/python -m pytest -q
```

用仓库自带的模拟数据试跑：

```bash
.venv/bin/baiquant sample-data --output-dir examples/data

.venv/bin/baiquant run \
  --prices examples/data/prices.csv \
  --stocks examples/data/stocks.csv \
  --fundamentals examples/data/fundamentals.csv \
  --events examples/data/events.csv \
  --start 2024-01-01 \
  --end 2024-01-10
```

## 接入真实数据

BaiQuant 不附带真实行情数据。你需要自己准备数据源。

如果使用 Tushare，可以先设置 token：

```bash
export TUSHARE_TOKEN="your-token"
```

再拉取数据到本地 SQLite：

```bash
.venv/bin/baiquant ingest tushare \
  --output data/tushare/baiquant.db \
  --start 20230101 \
  --end 20260708 \
  --adjust qfq \
  --write-mode append \
  --resume
```

生成盘后计划：

```bash
.venv/bin/baiquant live20k-2026 \
  --db data/tushare/baiquant.db \
  --as-of 2026-07-08 \
  --preset steady-20d \
  --cash 50000 \
  --plan-output data/paper/live20k_plan.csv
```

启动本地 Web UI：

```bash
.venv/bin/baiquant desk \
  --db data/tushare/baiquant.db \
  --host 127.0.0.1 \
  --port 8765
```

## 后续我想继续写什么

这只是 Codex 系列的第一篇。

后面我可能会继续记录：

- Codex 如何帮我选择 Tushare 和设计本地数据结构；
- 为什么很多看起来聪明的策略回测后并不好；
- AI 如何解释一只股票为什么被选中或被剔除；
- 真实账户里，策略和人性之间的冲突；
- 如何判断一段收益到底来自策略、行情，还是运气；
- Codex 在金融、产品、运营、研发之外还能产生什么价值。

对我来说，这个系列真正想回答的问题不是“AI 能不能炒股”，而是：

> 当我们把 AI 放进一个足够真实、足够复杂、反馈足够快的场景里，它能不能帮助一个人
> 更快地学习、判断、验证，并且承担更高难度的目标？

BaiQuant 是这个问题的第一个实验。
